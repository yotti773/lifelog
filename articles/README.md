# 記事シリーズ企画(目次案・運用ルール)

このディレクトリは、からだログの開発・減量実験を記事化するための**下書き置き場**。`docs/` の仕様書群とは役割が異なり、レビューチェックリストのドキュメント整合チェックの対象外とする(記事は公開時点のスナップショットであり、仕様の真実の情報源ではない)。

## 全体コンセプト

「個人開発で食事記録アプリを作った」ではなく、次の3本柱で書く:

1. **開発プロセス**: 仕様書駆動 + AIエージェント(Claude Code)で個人開発を回す実例
2. **設計の中身**: ローカルファースト、決定論的計算+AI言語化の分業、壊れる前提の外部連携
3. **n=1の物語**: 自作アプリで自分を計測し、10/31までに72kg→64kgを目指す減量実験(実データ付き)

## マネタイズとの関係(2026-07-16の整理)

この発信は**それ自体が収益源ではない**。note/Zennの有料記事・投げ銭は数千人規模の読者がいて初めて意味を持つため、現状(フォロワー実質0)では直接的なマネタイズ手段にならない。位置づけは「継続して発信することで読者・信頼という資産を積む」こと。将来この資産が使えるとしたら: (1) 読者規模が育てば有料記事・本が収益化する、(2) 実務価値の高いテーマでの登壇・執筆依頼につながる、(3) 「自分データの収集・分析ツール」を事業化する場合の初期顧客候補になる。**急いで収益化(有料化・宣伝の多用)を狙わず、淡々と継続することを優先する**(2026-07-16、本人決定)。

1. 今週の数字(週平均体重・前週比・平均摂取kcal・記録率・実測TDEE) — 週次レビュー画面のスクショ+転記

4. 来週やること(アプリが提案するアクションから1つ)

## 公開ライン(書く前に一度だけ決める)

アプリ設計で「日記本文はデフォルトでAIに送らない」と線を引いたのと同様に、公開範囲も先に決めておき、毎回悩まない。**以下は初回公開前に本人が確定させる**:

- [x] 体重・体脂肪率の実数を出す(2026-07-16決定。記事1で72kg→64kgを使用。物語の根幹のため)
- [x] 摂取カロリー・PFCの実数を出すか → **週平均のみ出す**(2026-07-17決定。日次の値・食事内容の詳細は出さない)
- [x] 日記本文・気分タグ → **本文は出さない**(固定)。気分タグの週次集計(絶好調・良い・眠い・不調などの日数分布)は**出す**(2026-07-17決定)
- [x] 血圧の実数を出すか → **出さない**(2026-07-17決定。要配慮度が高いデータのため)
- [ ] スプレッドシート・設定画面のスクショはシートID・APIトークン・メールアドレスをマスクする(固定)
- [x] リポジトリはpublic化済み(2026-07-16。シークレット履歴・Issue内の個人情報は点検済み)
- **下書きは公開前提で書く(2026-07-16決定)**: このディレクトリはリポジトリごと公開される。無料記事の下書き・執筆メモが公開で読めるのは許容(build in public)。**noteで有料化する記事の本文だけは、このリポジトリに置かない**(リポジトリ外で執筆する)
