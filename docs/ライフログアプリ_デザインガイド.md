# ライフログアプリ デザインガイド

作成日: 2026-07-01
関連ドキュメント: ライフログアプリ_要件定義書.md / ライフログアプリ_画面設計書.md

## コンセプト
「食欲とエネルギーのポップさ」。健康管理アプリにありがちな白×青緑の無機質な配色ではなく、食事記録アプリであることを活かした、元気で続けたくなる配色・タイポグラフィにする。主役は体重・カロリーの「数字」で、それを見るのが楽しくなることを目指す。

## カラーパレット

| 役割 | カラー名 | HEX | 用途 |
|---|---|---|---|
| 背景 | アイボリーホワイト | `#FFF8F0` | 画面全体の背景。純白より柔らかい印象 |
| プライマリ | コーラルオレンジ | `#FF6B4A` | メインボタン、＋(記録)ボタン、選択中タブ |
| セカンダリ | フレッシュミントティール | `#2EC4B6` | 体重グラフの線、カロリー進捗バー、健康・前進を示す要素 |
| アクセント | サニーイエロー | `#FFC145` | 目標達成・連続記録などのお祝い演出に限定使用 |
| テキスト(主) | チャコール | `#2B2B2B` | 本文・数字。純黒より柔らかい |
| テキスト(補助) | ウォームグレー | `#8C8C8C` | タイムスタンプ、補足ラベルなど |

※ アクセントのイエローは使いすぎると煩雑になるため、「達成した瞬間」だけに使う、という制約を持たせる。

### ダークモード(優先度低・将来対応)
| 役割 | HEX |
|---|---|
| 背景 | `#1E1E2A` |
| テキスト(主) | `#F0F0F0` |
| プライマリ/セカンダリ/アクセント | ライトモードと同じ色をそのまま使用(暗背景でも視認性・彩度を保てる色を選定済みのため) |

## タイポグラフィ

| 役割 | フォント | 用途 |
|---|---|---|
| 見出し・数字(体重/カロリー) | M PLUS Rounded 1c(丸ゴシック) | 体重・カロリーなど「主役の数字」に使用。丸みで親しみやすさを演出 |
| 本文 | Noto Sans JP | 通常のテキスト、リスト項目など可読性重視の箇所 |
| 補助・キャプション | Noto Sans JP(Light) | タイムスタンプ、メモ、補足情報など |

数字表示は大きめのサイズ・太めのウェイトで見せ、「今日の記録一覧」でも体重・カロリーの数値が一番目立つように強弱をつける。

## レイアウト方針

- カード形式のUI。角丸(16px程度)+ ふんわりした影で、ポップさと柔らかさを両立
- ボトムナビゲーションは選択中のタブに丸いピル形のインジケーターを表示
- 記録用の「＋」ボタンはプライマリカラーの円形フローティングボタンとして常時表示
- カロリー摂取の進捗は、ゲーム的な「太めの角丸プログレスバー」(セカンダリカラーで塗りつぶし)で表現し、数字だけでなく視覚的な達成感を出す

## シグネチャー要素:「ゴールバー」

推移画面のトップに、現在体重(72kg)→目標体重(64kg)の進捗を示す横長の**ゴールバー**を配置する。これがこのアプリの象徴的な要素になる。

```
[72kg]━━━●━━━━━━━━━━━━━━[64kg]
        現在値                10/31まであと◯日
```

- バーの塗り(現在地までの部分)はセカンダリのミントティール
- 目標到達時にはバー全体がアクセントのイエローに変化し、軽い達成演出(小さな紙吹雪アニメーションなど)を1回だけ表示する

## モーション(控えめに)

- 記録保存時: ボタンやカードが軽くバウンドする程度の小さなフィードバック
- 目標達成時(カロリー目標達成、体重目標達成など節目): 上記の紙吹雪演出のみ
- それ以外の派手なアニメーションは入れない(AIっぽい過剰演出を避け、記録という実用行為を邪魔しないため)

## 実装メモ
- Tailwind CSSのカスタムカラー(`tailwind.config.js`の`theme.extend.colors`)に上記パレットを登録して使用する
- フォントはGoogle Fonts経由で読み込み(`M PLUS Rounded 1c` / `Noto Sans JP`)
